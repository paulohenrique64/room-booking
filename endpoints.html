JWT Autenticação

🔐 Autenticação JWT

Obter Token: POST /api/token
Renovar Token: POST /api/token/refresh
Headers: Authorization: Bearer YOUR_TOKEN

🔐

Autenticação

POST /api/token Público
Descrição Obter token JWT com credenciais de usuário
Corpo da Requisição
{ "username": "admin", "password": "admin123" }
Resposta
{ "refresh": "...", "access": "..." }
POST /api/token/refresh Público
Descrição Renovar token JWT expirado
Corpo da Requisição
{ "refresh": "REFRESH_TOKEN" }
Resposta
{ "access": "NEW_ACCESS_TOKEN" }
🏢

Salas

GET /api/v1/salas Autenticado
Descrição Listar todas as salas ativas
Resposta
{ "count": 5, "results": [...] }
GET /api/v1/salas/{id} Autenticado
Descrição Obter detalhes de uma sala específica
Parâmetros id - ID da sala (inteiro)
GET /api/v1/salas/{id}/disponibilidade Autenticado
Descrição Verificar horários bloqueados para uma sala em uma data
Query Parameters data - Data no formato YYYY-MM-DD (obrigatório)
Exemplo
/api/v1/salas/1/disponibilidade?data=2026-05-22

Reservas

GET /api/v1/reservas Autenticado
Descrição Listar reservas (suas, se professor; todas, se admin)
Query Parameters Opcionais status - Filtrar por status (ativa, cancelada, finalizada)
data - Filtrar por data (YYYY-MM-DD)
sala - Filtrar por sala (ID)
GET /api/v1/reservas/minhas_reservas Autenticado
Descrição Listar todas as reservas do usuário autenticado
Query Parameters Opcionais status - Filtrar por status
data - Filtrar por data
GET /api/v1/reservas/proximas_reservas Autenticado
Descrição Listar próximas reservas do usuário nos próximos 30 dias
Retorna Apenas reservas ativas com status 'ativa'
POST /api/v1/reservas Autenticado
Descrição Criar nova reserva de sala
Corpo da Requisição
{ "sala": 1, "data": "2026-05-22", "hora_inicio": "14:00:00", "hora_fim": "16:00:00", "motivo": "Aula de Engenharia" }
Validações • Hora de fim > Hora de início
• Data não pode ser no passado
• Sem conflito com outras reservas
PATCH /api/v1/reservas/{id} Autenticado
Descrição Editar reserva existente (apenas criador ou admin)
Parâmetros id - ID da reserva (inteiro)
Campos Editáveis hora_inicio, hora_fim, motivo
POST /api/v1/reservas/{id}/cancelar Autenticado
Descrição Cancelar reserva existente com motivo auditado
Corpo da Requisição
{ "motivo": "Aula cancelada por feriado prolongado" }
Resposta
{ "sucesso": "Reserva cancelada com sucesso" }
DELETE /api/v1/reservas/{id} Autenticado
Descrição Deletar reserva do banco (apenas admin ou criador)
Parâmetros id - ID da reserva (inteiro)
